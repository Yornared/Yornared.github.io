Wizard Chess: Automatic Chessboard

This project is a fully automated chessboard that physically moves pieces using a CoreXY gantry system, stepper motors, and an electromagnet. The board communicates wirelessly with an ESP32-S3 microcontroller, which interprets moves from the Lichess API and translates them into precise motor commands.

This project was awarded: "Best student project 2024" at group T

Features

  • Play against AI or human opponents on Lichess while the board mirrors the moves in real time.
  • Accurate calibration with corner sensors and automatic re-homing.
  • Bluetooth connection for setup and control.

Components

Design
Built on a cost-effective CoreXY rail system, inspired by 3D printer mechanics for accuracy.
Electronics
Powered by stepper motor drivers, an electromagnet for moving pieces, and a custom-built chess clock with wireless communication via nRF24.
Software
A React-based web app connects the board to Lichess, tracks the game state, and generates movement scripts for the magnet. Special handling was implemented for knight jumps, castling, and captures.

My contributions

I programmed the motor control system on the ESP32-S3, implemented Bluetooth Low Energy and webserver communication with the React app, and co-designed and assembled the CoreXY rail system and board hardware.

UniGigs: Gamified Freelancing SaaS for Universities

UniGigs is a SaaS platform that enables freelancing services within university communities. It integrates gamification elements such as token-based transactions, login streak rewards, user levels, and leaderboards to drive engagement.

This project was awarded: "Most technical SaaS application 2025" at Group T

Features

  • Intuitive UI/UX designed with Figma, focusing on ease of use and engagement.
  • Dual-role system allowing users to act as both service providers and seekers.
  • Token-based currency system with real money conversion via Stripe integration.
  • Gamification features including login streaks, user levels, and leaderboards.
  • Real-time chat functionality and notifications for seamless communication.
  • Comprehensive profile management and gig tracking for users.

Technologies

  • Frontend: React with TypeScript
  • styling: Tailwond CSS and Flowbite.
  • Backend: Firebase for authentication, database management, and cloud functions.
  • Payments: Stripe API

My Contributions

  • Implemented communication to Firebase NoSQL database (initially planned with Dataconnect, but migrated due to beta limitations).
  • Implemented the My Posted Gigs and Schedule tabs for managing gigs.
  • Designed and built the UniGigs landing page.
  • Performed design cleanup to ensure consistency across views.
  • Actively contributed to brainstorming features and shaping the overall design direction.
  • Collaborated on UI/UX design.

Distributed Food Ordering System

This project is a distributed food ordering system that allows users to browse a meal catalog, pick and choose meal orders, and manage their cart. The system is built using a microservice architecture with separate services for the frontend, backend, and database.

Features

  • Browse the catalog to select meals for a box with or without account creation.
  • Database and supplier services replicated and deployed across the globe for consistency.
  • Ingredient availability caching for live shop updates.
  • Distributed 2PC transaction protocol across suppliers to ensure order validity.

Components

Frontend
React with TypeScript, deployed as an Azure Static Web App.
Backend
Azure PostgreSQL database/server, Azure VMs, with REST APIs enabling communication between services.
Networking
Azure subnets for secure communication between supplier service and database.
Build Tools
Maven and Spring Boot for project build, dependency management, and service orchestration.

My Contributions

  • Implemented Supplier services.
  • Orchestration service to coordinate the suppliers
  • 2PC protocol between orchestrator and suppliers.
  • REST apis between suppliers, orchestrator and shop
  • H2 caching of meal recipes and ingredient availabilities
  • Deployed all services using GitHub actions on azure datacenters using student credits.

C++ Game

A 2D game developed in C++ using QTCreator. The game is designed to be as extendable as possible, with a modular architecture that allows for easy addition of new features and levels.

Features

  • ASCII-based visualisation, overlay-based visualisation, player attack/death animations.
  • Multiple input methods, including WASD keys, arrow keys and command-based input
  • Autoplay and mouse input using A* pathfinding.
  • Multiple levels with different terrain, enemies, and obstacles.
  • Variety of enemy types (including ones that interact with and affect the environment), consumables like health ups, and player energy consumption based on status effects and terrain.

Components

Development Tools
QTCreator, valgrind, memcheck, gitlab, UML
Design patterns
MVC, command, registry, singleton

My contributions:

  • Implemented main player and enemy architecture, including input handling, collisions and the utility game classes.
  • implemented the graphical view including animations and image loading / spritesheet creation.
